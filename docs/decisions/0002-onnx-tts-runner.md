# 0002: ONNX TTS Runner Boundary

Date: 2026-07-10

## Decision

Takokit will not claim Piper ONNX TTS execution until both halves of the runner are implemented and verified:

1. Text frontend: text normalization, sentence splitting, phonemization, and Piper phoneme ID preparation.
2. ONNX backend: session loading, tensor construction, session execution, audio tensor conversion, and WAV writing.

The current `piper-lessac` path resolves verified model/config artifacts and parses the Piper config, then returns typed `piper_text_frontend_not_implemented`.

## Why

The Piper ONNX graph does not accept raw text. The archived Piper Python runtime phonemizes text, maps phonemes through `phoneme_id_map`, adds BOS/PAD/EOS IDs, then runs ONNX inputs such as phoneme IDs, lengths, and scales.

The historical Piper frontend uses `piper_phonemize`/eSpeak-style phonemization. `rhasspy/piper-phonemize` is archived, lists an MIT license, and notes that newer Piper development moved to a GPL runtime line. Downstream packages also document GPL/eSpeak licensing concerns. Takokit must not vendor GPL runtime logic without an explicit adapter and license boundary.

Kokoro ONNX is also not raw-text simple. The ONNX Community model card says token IDs are generated by converting text to phonemes and mapping those phonemes to IDs before ONNX execution. So Kokoro is not a safe shortcut unless Takokit verifies and packages its frontend artifacts too.

## Current Implementation

- `takokit pull piper-lessac` installs verified ONNX model/config artifacts.
- The ONNX runner resolves the installed content-addressed artifact paths.
- The ONNX runner parses the Piper JSON config.
- Speech execution returns `piper_text_frontend_not_implemented`.
- `takokit runner install takokit-onnx` initializes the runtime directory and records `runtime-installed`, not `ready`.
- `takokit runner doctor takokit-onnx` reports ONNX session capability as not verified and Piper frontend status as `piper_text_frontend_not_implemented`.

## Candidate Next Route

`piper-plus-g2p` is an MIT Rust crate that advertises eSpeak-free multilingual G2P and Piper-style encoding against a model `phoneme_id_map`. It is a candidate for the next run, but it must be validated against the Lessac config and expected phoneme IDs before it can make `piper-lessac` executable.
